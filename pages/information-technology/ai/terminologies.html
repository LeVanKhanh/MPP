AI Terminologies

A practical reference for AI, LLM, agentic AI, and organisational adoption concepts.

Updated: July 4, 2026

How to use this reference

Each term is explained through five practical lenses: what it is, why it exists, when it is appropriate, how to apply it, and a primary study source. The goal is not to treat every term as a solution; select the smallest, safest approach that solves the business problem.

  • Start with the problem: define the workflow, user, decision, risk, and measurable outcome.
  • Prefer simple controls: prompt design, structured outputs, and deterministic workflows often precede RAG, agents, or fine-tuning.
  • Design for evidence: production AI needs evaluation, monitoring, data controls, escalation, and accountable ownership.

Foundation and Model Concepts

Concepts used to understand how AI models receive information, generate outputs, and behave in production.

LLM — Large Language Model

Definition
A neural-network model trained on large volumes of language data to predict and generate text; many LLMs can also reason over code and structured inputs.
Why
It provides a flexible language interface for tasks that would otherwise require many narrow NLP models or hand-written rules.
When
Use it when the task is language-heavy, varies in wording, and tolerates probabilistic output with validation.
How to use it
Start with a clearly bounded task such as summarising support tickets. Define the input, output schema, success criteria, and human review path before expanding scope.

OpenAI developer documentation

NLP — Natural Language Processing

Definition
The AI discipline concerned with analysing, understanding, and generating human language.
Why
It addresses the gap between unstructured human language and software that requires explicit, structured data or commands.
When
Use it for classification, extraction, search, translation, summarisation, and conversational interfaces.
How to use it
For example, extract product, priority, and customer sentiment from inbound emails, then route the result to a workflow.

Generative AI

Definition
AI that produces new content such as text, images, audio, code, or structured drafts from learned patterns and provided context.
Why
It reduces the effort required to create first drafts, transform content, and interact with complex knowledge using natural language.
When
Use it when output is created or transformed rather than only predicted or classified.
How to use it
Use it to draft a release note from accepted work items, with source links and an approval step before publication.

OpenAI developer documentation

Prompting / Prompt Engineering

Definition
The practice of supplying instructions, context, examples, constraints, and an expected format to guide model output.
Why
It reduces ambiguity and improves task reliability without changing the underlying model.
When
Use it before fine-tuning whenever a task can be improved through clearer instructions, examples, or grounded context.
How to use it
Specify the role, goal, input boundaries, required JSON schema, refusal behaviour, and two or three representative examples; evaluate against a fixed test set.
Documents / source
OpenAI prompt engineering guide

Tokens

Definition
The units of text a model reads and generates. A token may be a whole word, part of a word, punctuation, or whitespace.
Why
They provide the operational unit for context limits, throughput, and most model usage pricing.
When
Consider tokens whenever designing prompts, sizing retrieved context, estimating cost, or diagnosing truncation.
How to use it
Log input and output token counts by use case; set limits for document chunks and reserve space for the model response.
Documents / source
OpenAI developer documentation

Context Window

Definition
The maximum amount of tokenized information a model can consider in one request, including instructions, conversation history, retrieved data, and the output budget.
Why
It constrains how much evidence and history can be processed at once; exceeding it can cause errors, truncation, or loss of important context.
When
Use context-window design whenever conversations, documents, or RAG results may become large.
How to use it
Keep stable instructions short, retrieve only relevant passages, summarize prior conversation turns, and reserve output tokens for the required answer.
Documents / source
OpenAI developer documentation

Inference

Definition
Running a trained model on new input to produce an output; this is distinct from training or fine-tuning the model.
Why
It is the production-time operation that turns a model into a usable feature and creates latency and usage cost.
When
Use the term when discussing serving architecture, performance, capacity, cost, or runtime evaluation.
How to use it
Instrument each request with model version, prompt version, latency, token usage, tool calls, and outcome signals.
Documents / source
OpenAI developer documentation

Temperature

Definition
A sampling parameter that controls how concentrated or varied a model’s next-token choices are; lower values tend to be more repeatable and higher values more diverse.
Why
It lets teams tune the trade-off between consistency and creative variation.
When
Use low temperature for extraction, classification, and deterministic business workflows; use a higher setting for ideation or varied drafts.
How to use it
For a contract-field extractor, start near zero and enforce a schema. For marketing ideation, test a moderately higher setting and evaluate diversity against quality.

OpenAI developer documentation

Multimodal AI

Definition
AI that can process or generate more than one modality, such as text, images, audio, video, or documents.
Why
It enables workflows where meaning is distributed across formats rather than contained only in text.
When
Use it when visual, spoken, or document-layout evidence is necessary to complete the task.
How to use it
Build an invoice assistant that accepts a PDF or photo, extracts line items, highlights uncertain fields, and asks for confirmation before posting data.

Knowledge, Retrieval, and Integration

Patterns and components used to ground AI outputs in current enterprise information and connect models to operational systems.

RAG — Retrieval-Augmented Generation

Definition
An architecture that retrieves relevant external information at query time and supplies it to a model as evidence for its answer.
Why
It addresses stale model knowledge, private organisational data, and the need for answers supported by sources.
When
Use it when answers must be based on changing policies, product documentation, tickets, or other governed knowledge.
How to use it
Ingest approved documents, chunk and index them, retrieve relevant passages per question, ask the model to cite only those passages, and return “not found” when evidence is insufficient.
Documents / source
LangChain retrieval and RAG guide

Embeddings

Definition
Numeric vector representations of data that place semantically similar items close together in a vector space.
Why
They make semantic similarity and retrieval possible even when two texts use different wording.
When
Use them for semantic search, RAG retrieval, clustering, recommendation, and duplicate detection.
How to use it
Embed document chunks and the user question with the same embedding model, retrieve the nearest chunks, then rerank before giving them to an LLM.
Documents / source
OpenAI embeddings guide

Vector Database

Definition
A database or search service optimized to store vectors and retrieve the nearest vectors by similarity.
Why
It provides scalable, low-latency semantic retrieval for embeddings that conventional keyword search alone cannot provide.
When
Use one when you need semantic search over a sizeable or frequently queried collection.
How to use it
Store embedding, document ID, chunk text, source URL, access-control metadata, and version. Filter by user permissions before returning candidate chunks.
Documents / source
LangChain retrieval and RAG guide

MCP — Model Context Protocol

Definition
An open protocol for connecting AI applications to external tools, resources, and prompt-based workflows through a standard client–server model.
Why
It reduces one-off integrations and gives AI applications a consistent way to discover and use governed capabilities.
When
Use it when several AI clients need access to the same internal tools or data sources, or when you want reusable integrations.
How to use it
Expose a narrowly scoped MCP server for a read-only knowledge search API; require authorization, validate tool arguments, log calls, and start with least-privilege access.
Documents / source
Model Context Protocol documentation

LangChain

Definition
An application framework for composing LLM-powered applications and agents with models, prompts, tools, retrieval, and integrations.
Why
It reduces boilerplate when an application needs orchestration across multiple AI components.
When
Use it when its abstractions improve speed and observability; do not introduce it for a single straightforward model API call.
How to use it
Build a small proof of concept with explicit prompt templates, retrieval, and tool boundaries; add tracing and tests before relying on autonomous execution.
Documents / source
LangChain documentation

AI Automation Workflow

Definition
A controlled orchestration of AI models, data sources, APIs, deterministic business rules, and approval steps that automates part of a business process.
Why
It turns isolated AI responses into repeatable business outcomes while preserving predictable system controls.
When
Use it for repetitive, high-volume processes where inputs, actions, ownership, and exception paths can be defined.
How to use it
For incident triage: classify the ticket, retrieve relevant runbooks, create a proposed diagnosis, and require an engineer to approve any production action.
Documents / source
Microsoft AI adoption guidance

Fine-tuning

Definition
Additional training of a pretrained model on selected examples to better perform a narrow task, style, or output pattern.
Why
It can improve consistency and reduce prompt length for stable, high-volume behaviours that prompting and RAG do not adequately solve.
When
Use it only after proving that prompt engineering, structured outputs, and retrieval are insufficient, and when you have high-quality representative data.
How to use it
Create a curated training and holdout evaluation set, establish a baseline prompt, fine-tune only for the measurable gap, and continually compare quality, cost, and safety.

Agents and Operational Controls

Terms for action-taking AI systems and the engineering controls required to make their behaviour safe, observable, and useful.

AI Agents / Agentic AI

Definition
Systems that use an AI model to pursue a goal through multiple steps, often selecting tools, using retrieved context, and adapting from intermediate results.
Why
They address tasks that require planning and action across systems rather than a single generated response.
When
Use them when the workflow genuinely needs dynamic tool selection or multi-step reasoning. Prefer deterministic workflows for fixed processes.
How to use it
Start with a constrained agent that can read data and propose actions. Define tool contracts, maximum steps, approval gates, stop conditions, and audit logs.
Documents / source
LangChain documentation

Agentic Behaviour

Definition
A behaviour pattern in which an AI system decomposes an objective, chooses next actions, observes results, and iterates toward completion.
Why
It enables flexible handling of tasks whose sequence cannot be fully predefined.
When
Use it for investigation, research, triage, and assisted operations where the action path varies but the boundaries are well defined.
How to use it
Use a planner–executor pattern: create a bounded plan, execute only approved tools, verify each result, and surface the final evidence and decision trail.
Documents / source
LangChain documentation

Autonomous Agents

Definition
Agents permitted to take actions with limited real-time human intervention within defined authority, budget, and safety boundaries.
Why
They can reduce manual work in stable, low-risk operations, but amplify errors if controls are weak.
When
Use only after a supervised version proves reliable and the action has reversible, low-impact failure modes.
How to use it
Allow automatic creation of draft responses or low-severity tickets first. Require explicit human approval for spending, access changes, or irreversible production actions.
Documents / source
Microsoft Responsible AI guidance

Guardrails

Definition
Technical and policy controls that constrain model inputs, outputs, tool use, and escalation behaviour.
Why
They prevent unsafe, unauthorized, irrelevant, or non-compliant behaviour that a general-purpose model may otherwise produce.
When
Use guardrails in every production AI feature, especially when user input, sensitive data, or system actions are involved.
How to use it
Combine input validation, content and policy checks, schema validation, retrieval permission filters, tool allowlists, rate limits, and human escalation rules.
Documents / source
OpenAI safety best practices

Fabrications / Hallucinations

Definition
Outputs that are incorrect, unsupported by available evidence, or invented while appearing plausible or confident.
Why
They are a core reliability risk because fluent language can hide factual uncertainty.
When
Plan for this in every generative AI use case; treat it as a system-design concern, not merely a model defect.
How to use it
Ground answers in retrieved sources, require citations, validate structured fields, provide an “I do not know” path, and measure unsupported-claim rates in evaluation.
Documents / source
OpenAI safety best practices

Bias

Definition
A systematic and potentially unfair skew in model outputs, data, or decision processes that affects groups, cases, or outcomes differently.
Why
It can create discriminatory, inaccurate, or unrepresentative outcomes and undermine trust.
When
Assess it when AI affects people, eligibility, prioritization, performance evaluation, access, or other consequential decisions.
How to use it
Define affected groups and harm scenarios, evaluate representative cases, use human review for consequential decisions, and document known limitations and mitigations.
Documents / source
Microsoft Responsible AI guidance

Reliability

Definition
The degree to which an AI system produces consistent, correct, safe, and usable outcomes across representative conditions.
Why
It makes an AI capability dependable enough to include in a business process rather than treating it as a novelty.
When
Use it as a release criterion for any production use case, especially when errors affect customers, money, security, or operations.
How to use it
Create an evaluation suite with normal, edge, adversarial, and policy-sensitive cases; release only when quality, failure handling, and monitoring targets are met.
Documents / source
Microsoft Responsible AI guidance

Latency

Definition
The elapsed time between a request and a useful response or completed action.
Why
It determines whether an AI interaction feels responsive and whether a workflow meets operational service levels.
When
Track it for interactive assistants, real-time automation, and any process with SLA commitments.
How to use it
Measure end-to-end latency separately from retrieval, model generation, tool calls, and post-processing; cache stable results and use smaller models for simple tasks.
Documents / source
OpenAI developer documentation

Machine Learning Lifecycle

Definition
The end-to-end discipline of defining a problem, preparing data, training or selecting a model, evaluating, deploying, monitoring, and improving it.
Why
It addresses the fact that model quality and risks change after deployment as data, users, and environments evolve.
When
Use this lifecycle for predictive ML and adapt its governance, evaluation, deployment, and monitoring practices for generative AI.
How to use it
Define owners and gates for data quality, evaluation, deployment, monitoring, incident response, and retraining or prompt/model updates.

No-code AI Development Tools

Definition
Platforms that enable users to assemble AI experiences through visual configuration rather than conventional software development.
Why
They broaden participation and speed up low-complexity solutions, while still requiring governance and lifecycle discipline.
When
Use them for prototypes, departmental workflows, and bounded solutions where platform controls meet security and integration needs.
How to use it
Create a governed template for data connections, prompt instructions, approval flows, testing, and ownership before allowing broad self-service creation.

Organisation, Governance, and Adoption

Business and operating-model concepts for adopting AI safely, measuring value, and scaling beyond isolated experiments.

AI Governance

Definition
The policies, roles, decision rights, controls, and evidence used to direct and oversee how an organisation develops, buys, deploys, and monitors AI.
Why
It aligns AI activity with business value, security, privacy, legal obligations, and responsible-use expectations.
When
Implement it before scaling beyond limited experiments or whenever AI handles organisational data, customers, or business decisions.
How to use it
Establish intake and risk tiers, approval criteria, an inventory, data-use rules, model and vendor review, release gates, monitoring, and incident management.
Documents / source
Microsoft AI governance guidance

AI Council

Definition
A cross-functional governance group that steers AI strategy, prioritizes initiatives, resolves risk decisions, and coordinates business, technology, security, legal, and people perspectives.
Why
It prevents fragmented decisions and ensures AI initiatives have accountable sponsorship and proportionate oversight.
When
Create one when AI initiatives span departments, shared data, material risk, or substantial investment.
How to use it
Assign a charter, decision rights, required functions, meeting cadence, intake criteria, and a transparent record of approvals, conditions, and follow-up actions.

Microsoft AI governance guidance

AI Champions

Definition
Employees who help colleagues learn, experiment responsibly, share practical patterns, and surface adoption barriers within their teams.
Why
They bridge central AI strategy and the daily realities of work, increasing adoption through trusted peer support.
When
Use champions during pilot and scale phases after baseline policy and enablement materials exist.
How to use it
Select volunteers across roles, train them in responsible use and escalation, give them approved scenarios, and collect feedback through a regular community of practice.

AI Adoption

Definition
The process by which people, teams, and the organisation incorporate AI capabilities into real work with measurable, sustained use.
Why
It focuses attention on workflow change and outcomes, not simply license provision or technical deployment.
When
Use adoption planning whenever introducing an AI assistant, platform, or internal solution.
How to use it
Choose a high-value workflow, establish a baseline, train users in task-specific practices, measure quality and time impact, and iterate using user feedback.

Scaling AI Adoption

Definition
Expanding proven AI use from pilots to repeatable, secure, supported use across teams or the organisation.
Why
It addresses the gap between a promising demonstration and sustainable operating capability.
When
Scale only after a pilot demonstrates measurable value, acceptable risk, clear ownership, and supportability.
How to use it
Standardize reference architectures, onboarding, data access, evaluation, support, monitoring, and cost controls; scale by use-case class rather than copying one pilot everywhere.

Responsible AI

Definition
The practice of designing, deploying, and operating AI to meet principles such as fairness, reliability and safety, privacy and security, inclusiveness, transparency, and accountability.
Why
It protects people and organisations from avoidable harm, misuse, opaque decisions, and loss of trust.
When
Apply it from use-case discovery through retirement, with additional rigor for high-impact or sensitive scenarios.
How to use it
Conduct impact assessment, define harm mitigations, test safety and fairness, document limitations, provide appropriate transparency, and retain human accountability.

Microsoft Responsible AI guidance

Ethical AI

Definition
AI use that aligns with moral, social, and professional norms such as fairness, privacy, human dignity, and avoidance of harm.
Why
It broadens decision-making beyond legal compliance and technical performance to the effects on people and society.
When
Use ethical review when a system changes access, opportunity, safety, privacy, autonomy, or how people are treated.
How to use it
Discuss stakeholders, power imbalances, foreseeable harms, and acceptable trade-offs; make the resulting principles enforceable through design and governance controls.

AI Risk Management

Definition
A continuous process for identifying, analysing, prioritising, mitigating, monitoring, and communicating AI-related risks.
Why
It provides a disciplined way to manage uncertainty across model behaviour, data, security, misuse, operations, and business impact.
When
Use it for every AI system, with controls proportional to impact and likelihood of harm.
How to use it
Maintain a risk register covering data, model, security, user misuse, legal, and operational risks; assign owners, controls, residual risk, and review dates.

Accountability in AI Systems

Definition
Clear assignment of responsibility for decisions, controls, outcomes, and remediation throughout an AI system’s lifecycle.
Why
It prevents responsibility gaps when a system involves business owners, developers, vendors, model providers, and users.
When
Define accountability before launch and whenever a system has customer, operational, or consequential impact.
How to use it
Create a RACI covering use-case approval, data stewardship, model configuration, release approval, monitoring, incident response, and final business decisions.

Centralized AI Inventory

Definition
A governed register of AI tools, models, agents, vendors, datasets, use cases, owners, risk tiers, and lifecycle status.
Why
It provides visibility needed for oversight, audit, risk response, portfolio decisions, and cost management.
When
Create it once multiple teams can procure, build, or deploy AI independently.
How to use it
Record purpose, owner, users, data classification, model/vendor, integrations, risk assessment, approvals, monitoring status, and retirement date; make registration part of procurement and release workflows.

Microsoft AI governance guidance

Security Risk

Definition
The possibility that AI use causes unauthorized access, data exposure, unsafe tool actions, prompt injection, model abuse, or other compromise of systems and information.
Why
AI expands the attack surface because natural-language input, retrieved content, and tools can influence system behaviour.
When
Assess it before connecting models to internal data, code, APIs, or actions, and continuously after deployment.
How to use it
Enforce least privilege, isolate secrets, validate and authorize every tool action, treat retrieved content as untrusted, log events, test for prompt injection, and maintain incident response procedures.

Cost Management for AI

Definition
The practice of forecasting, allocating, monitoring, and optimizing AI spending across models, tokens, infrastructure, tools, licenses, and support.
Why
It prevents successful prototypes from becoming unpredictable or unsustainable operating costs.
When
Implement it before production scale and whenever usage-based model or infrastructure costs are material.
How to use it
Tag requests by product and feature, set budget alerts, monitor cost per successful outcome, cap retries and context size, select models by task complexity, and review vendor commitments regularly.

Copilot

Definition
Microsoft’s product brand for AI assistants and agents embedded in Microsoft experiences; more generally, a copilot is an AI assistant that supports a user’s work.
Why
It brings assistance into familiar work surfaces and can reduce context switching when access to organisational data is appropriately governed.
When
Use it for productivity tasks such as drafting, summarising, searching, analysis, and guided work within supported Microsoft tools.
How to use it
Begin with a specific role workflow, apply data permissions and sensitivity controls, train users to verify output, and measure task-level value rather than general activity.

Copilot Extensibility — Extend

Definition
Adding domain-specific capabilities to a copilot through agents, connectors, plugins, or integrations so it can access approved knowledge and actions.
Why
It closes the gap between a general assistant and an organisation’s actual systems, data, and workflows.
When
Use extension when an existing copilot provides the right user experience but lacks a governed connection to your domain capability.
How to use it
Expose a narrow, secure API or connector with clear intent, authorization, output contracts, and audit logging; validate the business value before building a broad integration.

Organisation and Culture — AI Readiness

Definition
The skills, leadership behaviours, incentives, trust, governance habits, and change capacity that determine whether AI can be adopted productively.
Why
Technology alone does not change work; readiness determines whether people can use AI critically, responsibly, and consistently.
When
Assess readiness before large rollouts or when adoption stalls despite available tools.
How to use it
Survey skills and workflow barriers, train by role and use case, set leadership expectations, protect time for learning, celebrate verified value, and create channels for concerns and feedback.

AI Literacy

Definition
The baseline ability to understand what AI can and cannot do, use it critically, recognise risks, and apply organisational policies.
Why
It reduces misuse, overreliance, fear, and low-value experimentation while enabling better adoption decisions.
When
Provide it to all employees before or alongside broad AI access, with deeper training for builders and approvers.
How to use it
Teach model limitations, data handling, prompt practices, verification, bias, security, and escalation through realistic role-based exercises.

Shadow AI

Definition
Use of AI tools, models, or data-sharing practices outside approved governance, visibility, and controls.
Why
It creates unmanaged risks such as data leakage, inconsistent decisions, compliance gaps, and duplicate spending.
When
Address it whenever employees can easily access consumer AI tools or adoption outpaces formal enablement.
How to use it
Offer approved alternatives that are useful, make policies clear and practical, create an easy intake route for new needs, monitor proportionately, and focus on enablement rather than punishment alone.

Build vs. Buy

Definition
A strategic decision between developing an AI capability internally, purchasing a vendor product, or combining both through extension and integration.
Why
It balances differentiation, speed, control, maintenance burden, security, and total cost.
When
Make the decision after defining the business problem, required capabilities, data constraints, integration needs, and success measures.
How to use it
Buy commodity capability; build when the workflow or intellectual property is a durable differentiator; use a time-boxed proof of value and compare lifecycle cost, not just initial price.

AI Maturity Model

Definition
A staged framework for assessing an organisation’s present AI capabilities and defining the next practical improvements across strategy, technology, governance, people, and value management.
Why
It turns a broad ambition to “become AI-enabled” into a sequenced roadmap rather than disconnected initiatives.
When
Use it for portfolio planning, executive alignment, and periodic reassessment as adoption grows.
How to use it
Assess current state with evidence, identify the limiting capability, choose a small number of measurable next-stage improvements, fund them, and reassess on a regular cadence.

ROI / TCO for AI Initiatives

Definition
ROI measures value gained relative to investment; TCO captures the full lifecycle cost, including licenses, model usage, integration, security, governance, training, support, and change management.
Why
Together they prevent AI decisions based on attractive demos or headline subscription prices rather than sustainable business value.
When
Use both at intake, pilot evaluation, procurement, and scaling decisions.
How to use it
Define a baseline and target metric, quantify quality and time impacts, include all lifecycle costs, test assumptions in a pilot, and report realised—not only forecast—benefits.

Study sequence

  1. Learn the foundation: LLMs, tokens, context windows, prompting, inference, and temperature.
  2. Build reliable knowledge experiences: embeddings, vector databases, RAG, guardrails, and evaluation.
  3. Add action carefully: workflow automation, MCP, agents, approval gates, and security controls.
  4. Scale responsibly: governance, adoption, risk management, accountable ownership, cost management, and maturity planning.
