1. Copilot Chat vs Microsoft 365 Copilot

Criteria Microsoft 365 Copilot Chat Microsoft 365 Copilot (licensed)
Grounding Web Web + work data through Microsoft Graph
License Included with eligible Microsoft 365 subscriptions Paid add-on license
Embedded in Word, Excel, Outlook, and Teams No Yes
Researcher and Analyst No Yes
Agent using organizational data Requires metered pay-as-you-go usage Included, including custom agents
Copilot Control System and Analytics No Yes

Decision signal: Need an answer grounded in internal emails or files inside a Microsoft 365 app → choose Microsoft 365 Copilot with a license.

Exam trap: “Microsoft Copilot” without “365” generally refers to the consumer experience using a Microsoft account, not the enterprise Microsoft 365 product.

2. Researcher vs Analyst vs Standard Copilot Chat

Tool Use it when Output / characteristic
Standard Copilot chat Quick, lightweight work such as summarizing an email or drafting a short response Fast, optimized for speed
Researcher Combining multiple web and work sources with multi-step reasoning A structured report with citations
Analyst Analyzing quantitative data, large workbooks, and trends Data-science-style reasoning and numeric explanation

Quick distinction: “Large spreadsheet, find trends” → Analyst. “Synthesize many sources into a cited report” → Researcher.

Both Researcher and Analyst require Microsoft 365 Copilot licensing.

3. Build, Buy, or Extend

Choice Choose it when
Buy / use an existing product The need is common, and an available product meets it with appropriate governance.
Extend with a connector You only need to add knowledge from an external source such as Salesforce, ServiceNow, or Jira.
Extend with an agent You need specialized behavior, a workflow, or actions. Use declarative agents for low-code and custom-engine agents for pro-code scenarios.
Extend with an action or plugin Copilot needs to call an API and perform work in another system.
Build The need is differentiated or unique and cannot be met through buying or extending.

Rule: Need extra knowledge → connector. Need distinct behavior, workflow, or action → agent. Common need with an existing product → buy. Unique unmet need → build.

4. Responsible AI: Six Principles and Recognition Signals

  • Fairness: Treat groups fairly; watch for biased data and discriminatory outcomes.
  • Reliability and Safety: Evaluate before release, design fallbacks, and monitor after deployment.
  • Privacy and Security: Protect data with access control, encryption, and suitable retention.
  • Inclusiveness: Serve diverse users and contexts.
  • Transparency: “Why did AI produce this result?” Explain the basis and limitations.
  • Accountability: “Who is ultimately responsible?” A person or organization owns the outcome.

5. Trap Patterns and Exam Logistics

  • “Roll out quickly and fix later” or “skip security review” is almost always incorrect.
  • RAG is for knowledge that changes frequently; fine-tuning is for stable specialized behavior or output format.
  • Pay-as-you-go fits uncertain or variable usage; a commitment tier fits stable, predictable usage.
  • Measure adoption through active use in priority workflows and business outcomes, not the number of assigned licenses.
  • The scaled passing score is 700 out of 1,000. Most questions are generally available, and some may be preview questions. Coding is not required.

6. High-Priority Exam Review: Copilot, Extensibility, Licensing, and Responsible AI

Copilot versions

Option Best description Exam decision signal
Copilot Chat Web-grounded chat for general questions and public information. Choose it when the task does not require Microsoft 365 work-data grounding.
Microsoft 365 Copilot A licensed enterprise Copilot that can be grounded in work data through Microsoft Graph. Choose it for emails, files, meetings, chats, and work inside Microsoft 365 apps.

Memory cue: Public web information → Copilot Chat. Organizational work data through Microsoft Graph → Microsoft 365 Copilot with a license.

Microsoft 365 Copilot extensibility: buy, extend, or build

  • Buy: Use an existing Microsoft or third-party product when it already meets a common business need with suitable governance.
  • Extend: Add capability to Copilot rather than recreating it. Use connectors to bring in external knowledge; use agents for specialized behavior, workflows, and orchestration; use plugins or actions when Copilot must call an API or perform work in another system.
  • Build: Create a bespoke solution only when the need is differentiated, strategic, or cannot be met by buying or extending.

Decision rule: Knowledge → connector. Behavior or workflow → agent. API operation → plugin/action. Common solved need → buy. Unique unmet need → build.

License and subscription tiers

Offering Commercial pattern Use it when
Copilot Included experience with eligible Microsoft 365 subscriptions, licensed monthly Microsoft 365 Copilot, or metered pay-as-you-go consumption for applicable agent scenarios. Match the option to whether the user needs public web chat, Microsoft 365 work-data grounding, or variable agent usage.
Microsoft Foundry Pay-as-you-go consumption or commitment tiers. Pay-as-you-go fits uncertain or variable demand; commitment tiers fit stable, predictable usage.

Exam trap: Do not assume every Copilot capability is included in a Microsoft 365 subscription. Check whether the scenario requires a Microsoft 365 Copilot license or metered pay-as-you-go usage.

Responsible AI: six named principles

The list contains six principles:

  • Fairness
  • Reliability and safety
  • Privacy and security
  • Inclusiveness
  • Transparency
  • Accountability

Recognition cues: Bias or unequal treatment → fairness. Explainability → transparency. Ownership of outcomes → accountability. Secure data handling → privacy and security. Robust operation and safeguards → reliability and safety. Support for diverse users → inclusiveness.

8. Answers and Decision Rationale

  1. B. Internal work-data grounding inside an app requires licensed Microsoft 365 Copilot. Copilot Chat is web-grounded, while “Microsoft Copilot” without “365” is the consumer product.
  2. A. Web-grounded Copilot Chat is available with eligible Microsoft 365 subscriptions without the Copilot add-on.
  3. B. An organizational-data agent in Copilot Chat requires metered pay-as-you-go consumption through Copilot Studio billing, or a Microsoft 365 Copilot license.
  4. B. Quantitative analysis of large tables and trend detection maps to Analyst; Researcher focuses on multi-source report synthesis.
  5. A. Multi-source research, multi-step reasoning, and cited reports map to Researcher.
  6. A. Adding knowledge from an external source without custom behavior or actions maps to a connector.
  7. A. Specialized workflow plus system actions maps to a Copilot Studio agent, not a connector alone.
  8. B. Prefer buy, then extend, then build. Do not build a common capability that an approved existing product already provides.
  9. B. Explaining the basis or rationale of a decision is transparency.
  10. B. Ultimate responsibility for outcomes belongs to people or the organization: accountability.
  11. B. Fast-changing knowledge should be retrieved through RAG from governed sources. Fine-tuning is not intended for constant knowledge updates.
  12. B. Stable specialized behavior or output format, especially with quality examples, is a fine-tuning scenario.
  13. B. Start with minimum controls and a governed pilot before scaling. “Roll out now, fix later” is a classic incorrect choice.
  14. A. A commitment tier can optimize cost when usage is stable and predictable; PAYG suits uncertain or variable demand.
  15. A. Excessive trust in an AI recommendation without critical review is automation bias.
  16. A. Use identity-aware access and permission trimming. Do not copy data beyond existing access rights.
  17. A. Microsoft Foundry supports model discovery, evaluation, comparison, and controlled deployment.
  18. A. Image-based product or label detection is a computer-vision scenario for Azure Vision.
  19. B. Adoption is valuable use in priority workflows and measurable impact, not entitlement or launch activity.
  20. B. An AI council provides strategic direction, oversight, and coordination among business, IT, security, legal, data, and change stakeholders.

Final Revision Checklist

  • Internal work data in Word, Excel, Outlook, or Teams → licensed Microsoft 365 Copilot.
  • Research across sources with citations → Researcher; quantitative workbook analysis → Analyst.
  • External knowledge → connector; custom workflow or action → agent.
  • Frequently changing knowledge → RAG; stable specialized behavior → fine-tuning.
  • High-risk AI → stronger evaluation, safeguards, monitoring, and human oversight.
  • Adoption success → active priority-workflow use plus business outcomes.
